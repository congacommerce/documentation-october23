Conga Digital Commerce October '23

Upgrading the Digital Commerce Template from February '23 to Oct '23

This guide is designed to step you through the process of upgrading digital commerce template to consume SDK from February '23 release to Oct '23 release.

Table Of Contents

  • Updating the libraries
  • Updating npmrc file

Updating the libraries

You must point these packages to the latest version:

  • @congacommerce/core ^2303.x.x

  • @congacommerce/ecommerce ^2303.x.x

  • @congacommerce/elements ^2303.x.x

After updating these packages, verify if your package.json under dependencies reflects the latest version as below:


"@congacommerce/core": "^2303.x.x",

"@congacommerce/ecommerce": "^2303.x.x",

"@congacommerce/elements": "^2303.x.x",

...

Add "querystring" to package dependencies library

...

"querystring": "^0.2.1"

...

Add "@types/estree" to package dependencies library

...

"@types/estree": "1.0.0"

...

Updating npmrc file

To install SDK libraries as npm packages, you must update .npmrc file to specify the github repository path where these packages are hosted.

@congacommerce:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken={{GITHUB AUTH TOKEN}}

As this is a public github repository with read permissions, you don't need any access to install these packages.
